## 本次发现的问题 → 立即修复

### §52a — `ai_series_ocf_reset_project_episode` 漏清 scene tree

**现象**:OCF 跑完产生 scene tree 节点,reset 只清 shot_tree_refs 不清 scene
**根因**:`ai_series_one_click_film.php:614-620` 只 foreach `$shot_tree_refs`
**修复**:加 step 6.5 — `JSON_UNQUOTE(JSON_EXTRACT(meta_json,'$.v1_episode_ref'))` 找到 ep_X 下所有 scene tree,空 scene(0 shot 子节点)用 `ai_series_tree_delete_node` 级联删
**验证**:Phase 4 跑完 scene tree=0 ✅

### §52b — `ai_series_delete_episode` 漏删 episode tree

**现象**:`delete_episode` API 调完,episode 表行删除但 ai_series_tree 行残留
**根因**:`ai_series_data.php:4729 ai_series_delete_episode()` 只删 shot tree 不删 episode 自身 tree 和它的 boards/scene 子节点
**修复**:加级联删 episode tree + 直系子(boards/scene 等)+ 二级孙(scene 下的 shot)
**验证**:Phase 7 调完 episode tree 同步删除 ✅

### §52c — `ai_series_project_asset` 死引用兜底

**现象**:RS `delete_resource()` 真删 resource 后,ai_series_project_asset 残留死引用
**根因**:RS 不知道 ai_series 自家关联表
**修复**:加 hook `HookAi_seriesAllBeforedeleteresourcefromdb` 同步清 project_asset + episode_shot
**验证**:Phase 8 显示 project_asset_dead_refs = 0 ✅

### list_episodes API endpoint 参数名(无需改后端,前端用对)

`api.php?action=list_episodes` 用参数 `project_ref`(不是 `project`)
