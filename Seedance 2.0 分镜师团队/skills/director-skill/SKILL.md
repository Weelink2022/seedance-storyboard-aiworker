# 导演执行技能

[技能说明]
    剧本分析与导演讲戏技能。将剧本/梗概/分场大纲转化为：
    1. 导演讲戏本（按剧情点分节，每节包含元数据 + 导演阐述）
    2. 人物清单（用于服化道生成角色参考图）
    3. 场景清单（用于服化道生成场景参考图）

    核心理念：导演像给演员讲戏一样，把脑海中的画面完整描述出来。
    一个剧情点 = 一段完整叙事 = 一次 Seedance 2.0 生成任务。
    段落内可包含多组镜头（如蒙太奇、快速闪回），但叙事上必须是一个完整段落。

[文件结构]
    director-skill/
    ├── SKILL.md                                    # 本文件（技能流程说明）
    └── templates/

[执行流程]

    第一步：通读剧本
        - 读取用户提供的剧本/梗概/分场文本
        - 理解核心叙事线、人物关系、情感走向
        - 识别关键转折点和高潮

    第二步：拆解剧情点
        将剧本拆分为若干剧情点（P01、P02、P03...），每个剧情点是一个完整的叙事段落。

        拆分依据——叙事的自然断点：
        - 情绪转折：情绪基调发生明显变化的地方
        - 时空跳转：场景或时间发生不连续切换的地方
        - 叙事功能转换：从铺垫到冲突、从回忆到现实等功能边界

        一个剧情点内可包含多组镜头：
        - 蒙太奇序列：多个快闪画面组成一个叙事段落（如"十年废材生涯的记忆洪流"）
        - 连续场景：同一叙事目的下的多个镜头
        - 单镜头：简单段落用一个连续镜头讲完

        反向检查：
        - 如果两个相邻段落服务于同一叙事目的 → 应合并
        - 如果一个段落内包含两个独立的叙事目的 → 应拆分

    第三步：确定时长

        Seedance 2.0 单次生成 4-15 秒。根据段落的叙事密度选择时长。

        时长选择参考：
            | 时长 | 适用场景                                         |
            |------|--------------------------------------------------|
            | 5s   | 单镜头，一个动作或表情变化                       |
            | 8s   | 单镜头，一组连贯动作                             |
            | 10s  | 多镜头，一段包含镜头转换的叙事                   |
            | 15s  | 多镜头，完整的多场景叙事（如蒙太奇、连续切换）   |

        节拍密度约束（作用于每个连续镜头内，非整段）：
            底层规律：AI 视频生成中，每个连续镜头内 1 拍 ≈ 2.5 秒屏幕时间。节拍太密，动作会糊、跳、不自然。

            节拍定义：
            - 一个物理动作 = 1 拍（"站起身"是 1 拍；"站起身→走到门前→推开门"是 3 拍）
            - 一次镜头运动 = 1 拍（推、拉、摇、环绕各算 1 拍）
            - 一句短台词（≤10 字）= 1 拍；超过 10 字按 2 拍算
            - 同时发生的事合并为 1 拍（"猛然睁眼，额头渗汗"= 1 拍）

            多镜头段落的节拍分配：
            段落内的节拍分布在多个镜头切换中，每个镜头各自遵守密度约束。
            例：15 秒蒙太奇可含 5 个子画面，每个 2-3 秒、1-2 拍，总节拍远超单镜头上限。

        头尾安全区：
            每次 Seedance 生成的前 0.5 秒和后 0.5 秒为安全区，不放关键内容。
            开头用于场景建立，结尾用于收住或过渡。

    新增步骤：确定视觉风格（在讲戏之前完成）
        在输出文档头部编写视觉风格说明，作为该集全局视觉参数：
        - 整体风格（真人写实 / 3D CG / 国漫 / 韩漫 / 日漫 / 自定义描述）
        - 色彩基调（暖/冷/高饱和/低饱和等）
        - 光影方向（硬光/柔光/自然光/戏剧性布光）
        - 材质质感（胶片颗粒/光滑/粗糙/水墨等）
        - 画幅比例（16:9 / 9:16 / 21:9 等）

        此全局参数将传递给服化道和分镜师，确保全阶段视觉一致。

    第四步：为每个剧情点"讲戏"
        先规划镜头 → 再写导演阐述。

        镜头规划（在写导演阐述前先完成）：
        每个剧情点须规划该段的镜头演进链，分三列：
        - 景别演进：如"全景建立 → 中景对话 → 面部特写"
        - 角度变化：如"平视 → 过肩 → 近景"
        - 运镜变化：如"固定 → 缓推 → 固定"
        多镜头段落分行列出每组镜头的规划。

        然后再写导演阐述，阐述必须包含以下维度（不需要分条列举，用自然流畅的叙述融合在一起）：

        画面内容：谁在哪里、环境是什么样
        人物动作：具体的肢体动作、走位、表情变化
        台词/声音：说了什么话、什么语气、环境音
        镜头感觉：什么景别、镜头怎么运动、从哪到哪
        光影氛围：什么光源、什么色调、什么情绪

        讲戏原则：
        - 用具体的物理动作，不用抽象概念（"揉太阳穴" ✓，"表现疲惫" ✗）
        - 用连续的动作链，让画面"动起来"
        - 镜头运动要有明确方向（"从侧脸推向窗外" ✓，"镜头移动" ✗）
        - 光影描述要具体（"冷白荧光灯照在脸上" ✓，"灯光" ✗）
        - 情绪通过动作和环境传达，不要直接说"她很伤心"
        - 微表情设计（仅近景/特写镜头）：
            - 近景和特写镜头中，将微表情作为"微观动作链"来描述——有运动过程、有时间推移、有变化轨迹
            - 格式：触发反应 → 变化过程 → 最终状态，且与镜头运动/环境元素协同
            - 静态标签写法（不要用）："瞳孔微缩""眼角泛红""表情坚定"
            - 动态过程写法（正确示范）："呼吸突然屏住，瞳孔先是茫然地散开，随后急剧收缩聚焦，像猎人锁定了猎物"
            - 非近景/特写镜头不需要微表情描述
        - 无旁白（短剧红线）：短剧中**禁止使用旁白/画外音叙事**。所有叙事信息须通过台词、动作、画面元素和音效传达。角色内心独白应以角色开口说话（有口型动作）的方式呈现，而非以第三方叙述性"旁白"的形式。
        - AI 视频生成限制（现代都市/写实题材）：
            - 避免描述物理破坏场景（踹门木屑飞溅、砸碎显示器、玻璃碎裂、物品爆裂等）
            - AI 视频模型处理物理破坏类场景容易出现穿帮/闪烁/变形
            - 需要表现激烈冲突时，改用人物表情/动作/运镜/音效来传达
            - "门被猛然踹开，木屑飞溅" ✗ → "门被猛然推开，重重撞在墙上，发出一声巨响" ✓
            - 古装/奇幻/动作题材中的武打破坏可适当放宽（但需评估AI生成效果）
        - 避免漫画/动漫视觉符号（写实题材）：
            - "眼镜闪过一道冷光""镜片反光闪白"这类描述是动漫视觉语法（柯南式），AI 会将其理解为漫画特效帧，在写实画风中生成闪光帧，导致画风突变
            - 改用真实布光中的物理光学现象来替代
            - "银边眼镜闪过一道冷光" ✗ → "镜框边缘反射着天花板的暖色筒灯光点，随着他转头的动作，光点从镜框前端滑向后端" ✓

        多镜头段落的额外要求：
        当一个剧情点包含多组镜头时，还需描述：
        - 镜头之间的切换方式（硬切、叠化、闪白、画面碎裂等）
        - 子画面之间的节奏（越来越快、均匀、先慢后快等）
        - 整段的叙事弧线（如"从希望到绝望的递进"）

    第五步：提取人物清单
        从剧情中筛选需要生成参考图的人物，只记录视觉外观信息：
        - 人物名
        - 年龄
        - 外观关键词（五官、发型、体型、服装等）

        筛选标准（满足任一即列入清单）：
        - 本集出现在 2 个及以上剧情点中的人物
        - 虽然本集只出现 1 次，但根据已有剧本判断后续集数会再次出场的人物

        不列入清单的人物（群演/一次性配角）：
        - 本集只出现 1 次且不会再出场的角色（如路人、围观群众、背景人物）
        - 这些人物不进清单，但在讲戏本的导演阐述中照常描述其外观和动作
        - 描述要足够具体，让分镜师能直接在提示词中用文字描写（如"尖脸眯缝眼的白袍学员"而非"一个路人"）

        注意：
        - 不需要情绪弧线，参考图是静态角色设定图
        - 人物清单 = 需要参考图的角色列表，服化道会为清单中的每个角色设计提示词

    第六步：提取场景清单
        从剧情中提取所有关键场景，只记录视觉环境信息：
        - 场景名
        - 时间（白天/夜晚/傍晚等）
        - 光线/色调
        - 氛围关键词
        - **角色着装要求**：该场景中各主要角色的着装（如"林薇：职业西装 / 母亲：家居服"），此列为变体提示词设计的源头输入

    第七步：素材比对（ep02+ 时执行）
        如果 assets/character-prompts.md 和 assets/scene-prompts.md 存在，读取已有人物和场景，为本集人物清单和场景清单中的每一项标注素材状态：

        判断规则：
        - 同一人物/场景未变化 → 标注"复用"
        - 新人物/新场景 → 标注"新增"
        - 同一人物外观变化（换装、老化、受伤等）→ 标注"变体"
        - 同一场景状态变化（季节、时间、破坏等）→ 标注"变体"

        ep01 或提示词文件不存在时，所有项标注"新增"。

    第八步：输出
        按照 templates/director-analysis-template.md 定义的格式输出。
        人物清单和场景清单中须包含"素材状态"列，服化道据此判断哪些需要设计。

[质量标准]
    - 每个剧情点的导演阐述至少 3-5 句话，有足够的视觉信息
    - 讲戏内容必须让分镜师能直接提取出运镜、动作、台词、光影
    - 剧情点之间要有叙事推进，不能重复信息
    - 人物清单和场景清单的外观关键词要足够具体，服化道能据此设计提示词
