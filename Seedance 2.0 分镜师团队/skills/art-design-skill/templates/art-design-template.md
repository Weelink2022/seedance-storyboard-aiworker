## 场景道具提示词格式

文件：assets/scene-prompts.md

每个场景独立生成一张多宫格图，展示该场景的不同视角。

普通场景：2×2 四宫格（1个全景建立 + 3个不同角度/焦距的画面）
重要/复杂场景：3×3 九宫格（多角度全覆盖）

```markdown
# 场景道具提示词

## [场景名]（epXX [集数] — 四/九宫格）

请生成一张 [2×2 / 3×3] 宫格布局的电影场景环境图像。所有格子代表**[场景名]**同一个场景的不同视角，风格统一、光影逻辑自洽。请按照以下规范生成。

### 视觉规范
整体风格：[描述整体美术风格]
色彩基调：[描述该场景的色彩倾向]
材质质感：[描述关键材质表现要求]

### 宫格布局
[根据场景数量标注方向]
第1行：格1-2 / 第2行：格3-4（四宫格）
或
第1行：格1-3 / 第2行：格4-6 / 第3行：格7-9（九宫格）

### Panel Breakdown（视角拆解）

格1——【[场景名] — 全景建立】
视角：[完整的叙事描述，包含该场景的地点和核心空间布局，从远距离展示全貌]
重点：[核心视觉元素，色调基调]

格2——【[场景名] — [视角名称]】
视角：[从一个特定角度/焦距展示场景的关键区域]
重点：[该视角下的核心视觉元素]

格3——【[场景名] — [视角名称]】
视角：[从另一个角度或不同景别描述]
重点：[核心视觉元素]

格4——【[场景名] — [视角名称]】
视角：[从细节或特殊角度描述]
重点：[核心视觉元素]

（四宫格4格，九宫格9格，多出的格子按更多视角补充）

（每个场景独立一节，按场景清单顺序排列）
```

## 输出规范

- 中文提示词
- 提示词采用叙事描述式，不要用关键词堆叠式
- 直接输出完整提示词，不要逐条解释设计理由
- 每个提示词必须完整可用，用户可以直接复制到文生图工具生成图片
- 每个条目标注创建集数（如 ep01 新增），便于追溯
- 如果是变体设计（如同一人物的冬装版），需在提示词中注明与原版的关联和差异
- 复用项不重复写入，只为"新增"和"变体"项编写提示词
- 场景宫格中所有格子必须保持风格统一，光影逻辑自洽
- 场景顺序按照导演分析中的场景清单排列
